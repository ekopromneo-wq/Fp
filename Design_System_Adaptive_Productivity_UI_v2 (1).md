# Дизайн-система мобильного приложения: «Adaptive Productivity UI»
**Версия:** 1.0 — 07.07.2026  
**Применение:** деловые / продуктивностные приложения (Stenogram, CRM, протоколы, задачи)  
**Платформы:** iOS 18+, Android 15+, WebView (PWA)

---

## 1. Философия

Интерфейс не отображает функции — он **предугадывает намерение**.  
Пользователь открывает приложение не «чтобы посмотреть», а «чтобы сделать».  
Каждый экран строится по принципу **«0→1 за 1 жест»** — от намерения к результату не более чем в одно действие.

> **Мантра:** *Меньше выбора. Больше контекста. Нулевое трение.*

---

## 2. Платформенные принципы (7 трендов 2026)

### 2.1. AI-First Adaptive UI
Интерфейс генерируется под контекст, а не рисуется статично.

| Контекст | Что показывает система |
|----------|------------------------|
| 09:00, календарь «Встреча через 20 мин» | Блок «Быстрая запись» на весь экран |
| 18:00, 3 несогласованных протокола | Блок «На согласовании» + кнопка «Отправить всё» |
| Вход из машины / шаттла | Крупные кнопки, голосовой ввод по умолчанию, нет мелкого текста |
| Вход с планшета / десктопа | Плотная таблица, боковая панель, drag-and-drop |

**Правило:** верхняя треть экрана — **прогноз системы**; нижняя треть — **действие пользователя**; середина — **контекстные детали**.

### 2.2. Тихий минимализм + Functional Density
- **12–17 pt** базовый шрифт (крупнее, чем в 2023, — aging population + mobile-first).
- **Карточки с глубиной** (glassmorphism 2.0): слои, размытие фона, тени без жёстких границ.
- **1–2 главных действия** на экране. Всё остальное — контекст (превью, статусы, теги, прогресс).
- **Плотность 60–72 dp** между строками в списках (компактно, но не тесно).

### 2.3. Bottom-Heavy Navigation
Весь управляющий интерфейс смещён в зону большого пальца.

- **Bottom Sheets** вместо полноэкранных переходов (уменьшает когнитивную нагрузку).
- **Floating Action Bar (FABar)** — раскрывающаяся панель из 3–5 действий, привязанная к правому нижнему углу.
- **Жесты > Кнопки:**
  - Свайп вправо — «Согласовать / Выполнить»
  - Свайп влево — «На доработку / Архив»
  - Pull-to-Action (не просто обновление, а «Создать из шаблона»)

### 2.4. Voice-Visual Hybrid
Голос — не отдельный режим, а **первичный канал ввода**.

- **Voice-first input:** удержание нижней кнопки → запись → расшифровка в реальном времени поверх интерфейса (transcription overlay).
- **Bimodal feedback:** звук + haptic + визуальная пульсация (например, при записи протокола телефон вибрирует в ритме речи).
- **Voice-to-Structure:** после остановки записи AI предлагает структуру («Распознано 3 спикера. Создать протокол?»).

### 2.5. Dark Mode as Default + OLED-оптимизация
- **Базовая тема:** тёмно-графитовый (`#0F1419`) вместо чистого чёрного (`#000000`) — убирает смазанные границы OLED.
- **Глубина через цвет:**
  - Фоновый слой — графитовый
  - Поднятый слой — глубокий синий (`#1A2332`)
  - Активный элемент — акцентный цвет (тиффани / брендовый)
- **Auto-contrast:** элементы регулируют яркость под True Tone.

### 2.6. Micro-interactions с «физикой»
- **Elastic scrolling:** список «пружинит» на конце.
- **Progressive disclosure:** кнопка при нажатии плавно трансформируется в карточку с деталями (shared element transitions).
- **Haptic morse:** короткие вибро-шаблоны:
  - Успех: «тик-тик»
  - Ошибка: «бррр»
  - Ожидание: пульсация 2 Гц

### 2.7. Accessibility-First (регуляторный стандарт)
- **Dynamic Type:** приложение не ломается при системном увеличении шрифта до 200 %.
- **Focus indicators:** чёткие рамки фокуса для клавиатурной/голосовой навигации.
- **Cognitive load guard:** если AI определяет, что экран перегружен, предлагается «Упрощённый режим» (скрывает второстепенное).

## 3. Спецификация компонентов

### 3.1. Контекстная лента (Context Feed)
Главный экран. Не меню, а **лента событий**.

```
┌─────────────────────────────┐
│  ┌─────────────────────┐    │
│  │ Встреча через 20 мин │    │ ← Прогноз системы (AI)
│  │ [Начать запись]      │    │
│  └─────────────────────┘    │
│                             │
│  Сегодня                    │
│  ┌─────────────────────┐    │
│  │ Протокол от 07.07   │    │ ← Контекстные детали
│  │ Статус: на согласовании│   │
│  └─────────────────────┘    │
│                             │
│  ┌─────────────────────┐    │
│  │ 3 задачи от Ксении  │    │
│  └─────────────────────┘    │
│                             │
│     [🎤]                    │ ← Действие (Bottom)
└─────────────────────────────┘
```

**Поведение:**
- Если в календаре встреча → верхний блок = «Запись».
- Если нет встреч → верхний блок = «Последние протоколы».
- Если вход в 22:00 → верхний блок = «Завтра: 2 встречи».

### 3.2. Карточка протокола (Protocol Card)
- **Высота:** 120–160 dp (адаптивно под контент).
- **Структура:**
  - Левая треть: аватар/иконка + цветовой индикатор статуса.
  - Середина: заголовок (2 строки), мета (дата, спикеры).
  - Правая треть: чипы действий (свайп или кнопки).
- **Depth:** фоновое размытие (backdrop blur 12 px), тень `0 4 24 rgba(0,0,0,0.12)`.

**Swipe-actions:**
- Вправо (зелёный градиент) → «Согласовать» + haptic «тик».
- Влево (оранжевый градиент) → «На доработку» + haptic «брр».
- Удержание → «Поделиться / Экспорт в Word».

### 3.3. Панель записи (Voice Panel)
- **Позиция:** bottom sheet, высота 280 dp.
- **Состояния:**
  1. *Idle:* крупная кнопка «Удерживайте для записи» (80×80 dp, тиффани).
  2. *Recording:* пульсирующий круг + waveform (живая волна) + распознаваемый текст в реальном времени.
  3. *Processing:* индикатор прогресса + «Распознаём спикеров…».
  4. *Done:* превью структуры + кнопки «Протокол / Задачи / Отмена».

### 3.4. Bottom Navigation (FABar)
```
┌─────────────────────────────┐
│                             │
│                             │
│                             │
│          ┌──────┐           │
│          │  +   │           │ ← Primary FAB (тиффани)
│          └──┬───┘           │
│     ┌───────┼───────┐       │
│     │ Запись│Загруз │       │ ← Secondary actions
│     │  🎤   │  📎   │       │    (появляются при тапе)
│     └───────┴───────┘       │
└─────────────────────────────┘
```

## 6. Паттерны взаимодействия

### 6.1. «Запись → Структура → Протокол»
1. Пользователь удерживает FAB → запись.
2. Система показывает waveform + live-transcription overlay.
3. Отпускание → AI анализирует (3–5 сек).
4. Bottom sheet раскрывается: «Распознано 3 спикера. Создать протокол?»
5. Тап → переход в редактор с пре-заполненной структурой.

### 6.2. «Согласование в один жест»
1. Лента протоколов.
2. Свайп вправо по карточке → зелёный градиент + иконка ✓.
3. Отпускание → карточка улетает вверх, появляется toast «Протокол согласован».
4. Haptic: «тик-тик».

### 6.3. «Контекстный поиск»
1. Пользователь тянет ленту вниз (pull-to-action).
2. Появляется поле поиска + 3 AI-подсказки:
   - «Протоколы за эту неделю»
   - «Задачи от Ксении»
   - «Просроченные согласования»
3. Тап на подсказку → мгновенная фильтрация.

## 7. Голосовой интерфейс (VUI)

- **Trigger:** удержание FAB или голосовая команда «Запиши протокол».
- **Feedback:**
  - Визуальный: waveform + transcription overlay.
  - Тактильный: лёгкая пульсация в ритме речи (2–4 Гц).
  - Аудиальный: короткий «щелчок» при старте/стопе.
- **Error recovery:** если распознавание неуверенно → система подсвечивает сомнительные фрагменты жёлтым, предлагает тап для правки.

---

## 8. Accessibility

- **Minimum touch target:** 48×48 dp (Android), 44×44 pt (iOS).
- **Color contrast:** ≥ 4.5:1 для Body, ≥ 3:1 для крупного текста.
- **Screen reader:** все карточки имеют grouped accessibility label («Протокол встречи от 7 июля, статус на согласовании, двойной тап для открытия»).
- **Reduce motion:** при включении убираются пружинные анимации, остаются только fade + scale (0.95→1).
- **Cognitive load guard:** если на экране >7 карточек, появляется chip «Показать 3 важных» — AI фильтрует по релевантности.

## 9. Примеры экранов (описание)

### Экран «Главная» (Context Feed)
- Верх: Display-текст «Доброе утро» или контекстный блок (см. §3.1).
- Середина: вертикальная лента карточек (Protocol Cards) с группировкой по датам.
- Низ: FABar + мини-индикатор «3 задачи на сегодня».

### Экран «Запись» (Voice Panel)
- Полноэкранный bottom sheet.
- Центр: пульсирующий круг (тиффани) + waveform.
- Низ: live-transcription (2 последние строки).
- Фон: затемнённый `bg-overlay`.

### Экран «Протокол» (Reader / Editor)
- Верх: sticky header с заголовком + чипы спикеров (цвета).
- Середина: структурированный текст (спикеры, реплики, выделенные задачи).
- Низ: action bar («Согласовать / Отправить в Word / Добавить задачу»).

## 11. Контрольный чек-лист

- [ ] Все первичные действия доступны одной рукой (в зоне большого пальца).
- [ ] Голосовой ввод даёт визуальный feedback в реальном времени.
- [ ] Тёмная тема — default; светлая — опция.
- [ ] Контраст текста ≥ 4.5:1 на всех фонах.
- [ ] При 200 % системном шрифте интерфейс не ломается (скролл, переносы, скрытие второстепенного).
- [ ] Каждое действие имеет haptic-отклик.
- [ ] AI-лента показывает не более 1 прогнозного блока + 3–5 контекстных карточек.

## 12. Референс-дизайн: Smart Assistant Mobile App (адаптированный под тиффани)

**Оригинал:** https://dribbble.com/shots/26840430-Smart-Assistant-Mobile-App-UI-UX-Design  
**Автор:** Dhanyashree  
**Платформа:** Dribbble

### Почему выбран
Этот шот — лучшая визуализация раздела «AI-First Adaptive UI» и «Контекстная лента». Futuristic dark UI с soft-градиентами, glowing-эффектами и smart chat interface. Адаптирован под цветовую систему Stenogram.

### Адаптация цветовой схемы (оригинал → тиффани)

| Элемент оригинала | HEX оригинала | HEX адаптации (тиффани) | Примечание |
|-------------------|---------------|-------------------------|------------|
| Primary accent | `#00D4FF` (cyan) | `#81D8D0` | Основной тиффани |
| Glowing gradient start | `#00E5FF` | `#A7E8E0` | Светлый тиффани |
| Glowing gradient end | `#0080FF` | `#5BC4BA` | Тёмный тиффани |
| AI-message bubble | `rgba(0, 212, 255, 0.15)` | `rgba(129, 216, 208, 0.15)` | Glassmorphism с тиффани |
| User-message bubble | `rgba(255, 255, 255, 0.08)` | `rgba(248, 250, 252, 0.08)` | Нейтральный glass |
| Status indicator (online) | `#00FF88` | `#4ADE80` | Сохраняем зелёный success |
| Recording pulse | `#FF3366` | `#FB923C` | Warning/attention orange |

### Ключевые паттерны для переноса в Stenogram

1. **AI-лента (Context Feed):**  
   - Сообщения AI появляются с левой стороны с soft-glow тиффани.  
   - Каждый блок имеет `backdrop-blur(12px)` + `bg-overlay`.  
   - Аватар AI — круг с градиентом `linear-gradient(135deg, #81D8D0, #5BC4BA)`.

2. **Voice input overlay:**  
   - При удержании FAB — экран затемняется (`bg-overlay`).  
   - Центральный пульсирующий круг: внешнее свечение `box-shadow: 0 0 40px rgba(129, 216, 208, 0.4)`.  
   - Waveform — линии тиффани с градиентом по высоте.

3. **Smart suggestions chips:**  
   - Горизонтальный скролл под AI-ответом.  
   - Чипы с border `1px solid rgba(129, 216, 208, 0.3)` + фон `rgba(129, 216, 208, 0.08)`.  
   - Active state: фон `rgba(129, 216, 208, 0.2)` + glow.

4. **Bottom sheet (панель записи):**  
   - Drag-handle тиффани (`#81D8D0`), 4px высотой, 40px шириной.  
   - Фон sheet: `bg-elevated` + top-border `1px solid rgba(129, 216, 208, 0.1)`.  
   - Закрытие — свайп вниз с elastic-эффектом.

## 13. Сторонние инструменты для сложных элементов

Все сложные визуальные элементы — анимированные кнопки (например, кнопка записи с waveform и пульсацией), прелоадеры, фоновые эффекты (градиенты, частицы, glow), анимированные переходы и микро-интеракции — могут быть разработаны и отрендерены в сторонних приложениях и загружены в проект в виде внешних ассетов.

### 13.1. Допустимые форматы

| Формат | Применение | Ограничения |
|--------|------------|-------------|
| **Lottie (JSON)** | Векторные анимации: иконки состояний, прелоадеры, кнопки FAB, чек-анимации | Размер ≤ 500 КБ; не более 3 одновременных инстансов на экране |
| **Rive (.riv)** | Интерактивные анимации с состояниями: кнопка записи, waveform, реакции на свайп | Runtime-зависимость; предпочтительно для кнопок с переходами Idle→Recording→Processing |
| **After Effects → Bodymovin/Lottie** | Сложные анимации переходов, splash-экраны, фоновые частицы | Растеризовать сложные эффекты; избегать heavy-mesh |
| **SVG + CSS-анимация** | Лёгкие glow-эффекты, пульсации, бесконечные лупы | Только для простых паттернов; проверять производительность на low-end устройствах |
| **WebP / PNG sequence** | Растровые анимации, если вектор неосуществим | Sequence ≤ 120 кадров; альфа-канал обязателен |

### 13.2. Правила интеграции

1. **Цветовая согласованность.** Анимации должны использовать токены цветовой системы (§5). При экспорте из сторонних редакторов применять глобальные swatches / тиффани-палитру:
   - Primary: `#81D8D0`
   - Light: `#A7E8E0`
   - Dark: `#5BC4BA`
   - Success: `#4ADE80`
   - Warning: `#FB923C`

2. **Прозрачность.** Все ассеты с анимацией поверх интерфейса должны поддерживать альфа-канал (прозрачный фон). Запрещено использовать цветной фон внутри анимации — фон задаётся системой.

3. **Размеры и оптимизация.**
   - Lottie JSON: ≤ 500 КБ
   - Rive файл: ≤ 800 КБ
   - PNG sequence (zip): ≤ 2 МБ
   - SVG: ≤ 50 КБ
   - При превышении — разбивать на части или заменять упрощённой версией.

4. **Accessibility и `prefers-reduced-motion`.**
   - Для каждого анимированного ассета создавать статичный fallback (первый/последний кадр или упрощённая версия).
   - При включении системной настройки «Уменьшить движение» анимация заменяется на fade/scale или статичный кадр.
   - Haptic-отклики (§2.6) остаются независимо от визуальной анимации.

5. **Платформенные рекомендации.**
   - **iOS:** предпочтительно Lottie-iOS или Rive-iOS; для сложных переходов — Core Animation с pre-rendered assets.
   - **Android:** Lottie-Android, Rive-Android; для прелоадеров — AnimatedVectorDrawable как lightweight-альтернатива.
   - **Web / PWA:** Lottie-web, Rive-web (Canvas/WebGL); для фонов — CSS-анимации или lightweight WebGL.

### 13.3. Примеры элементов для внешней разработки

- **Кнопка записи (FAB):** Rive-анимация с состояниями Idle → Press → Recording (waveform + пульсация) → Processing → Done.
- **Прелоадер AI-обработки:** Lottie-анимация в виде мягкого светящегося кольца тиффани-цвета.
- **Фон Voice Panel:** CSS/SVG-анимация мягкого градиентного glow при записи.
- **Swipe-индикаторы:** Lottie-иконки для состояний «Согласовать» (галочка) и «На доработку» (стрелка).
- **Splash / Onboarding:** After Effects → Lottie, адаптированная под тёмную тему и тиффани-акценты.

### 13.4. Контроль качества ассетов

- [ ] Анимация воспроизводится плавно (≥ 60 FPS) на mid-range устройствах (iPhone 13 / Pixel 6).
- [ ] Цвета совпадают с токенами после экспорта (проверка в sandbox).
- [ ] Fallback-версия подготовлена и протестирована с `prefers-reduced-motion`.
- [ ] Размер ассета не превышает лимиты (§13.2.3).
- [ ] Ассет корректно отображается в тёмной и светлой темах (применяются токены, а не hardcoded-цвета).
