## Эпик 14: Telegram standalone-бот (US-14.1) — 16.07

Коммит `88afe2d`, выкачен (миграции 023, 024 на проде). Пользователь шлёт боту голосовое/аудио/видео/ссылку в личном чате — получает протокол с задачами обратно, без открытия приложения.

- Миграция `024`: `telegram_link_codes` (код привязки, 15 мин), `telegram_chat_links` (bot_key+chat_id → user_id).
- Привязка: приложение выдаёт 6-значный код (POST /api/telegram/link-code), пользователь шлёт боту, поллер связывает. Код принимается **первым** даже для уже привязанного чата — пересвязка на другой аккаунт (нашёл этот баг в ходе теста: `resolveLinkedUser` перехватывал `/start КОД` раньше).
- Приём (`telegramIntake.js`, тот же getUpdates-поллер, что и кнопки задач US-11.3; `allowed_updates` += 'message'): voice/audio/video/video_note/document(audio,video)/URL. Файл идёт **тем же путём**, что загрузка из приложения (createRecording source=telegram_bot → attachRecordingAudio → enqueue) → история бота сразу в приложении. Только личные чаты.
- Результат обратно в чат: `notifyRecordingEvent` на done/failed для source=telegram_bot шлёт резюме+решения+задачи+ссылку (или сообщение о сбое).
- Лимит Telegram (критерий №2): Bot API отдаёт ботам ≤20 МБ; сверх — «слишком большой, загрузите через приложение».
- UI: блок «Бот-помощник» в настройках Telegram с кнопкой выдачи кода.
- **Проверка** — фальшивый Bot API (расширен: `/__message`, `/__set-file`, `/file/bot.../`, getFile), обработка аудио настоящая (OpenRouter): 14/14 сквозных + 3 UI. Оба критерия приёмки вживую: голосовое (сгенерирован валидный WAV) → done → «Готово: … Резюме … Задачи» в чат; файл 25 МБ → «слишком большой» без создания записи.
- Живой Bot API не проверялся — протокол эмулирован по документации; первая отправка настоящему боту может вскрыть расхождения (getUpdates offset, getFile, лимиты).

**Осталось из MVP**: эпик 16 (аккаунт: настройки, согласие на запись, доступ/хранение/удаление, Outlook-календарь), 17 (каталог исключений), 18 (устройства/доступность/поддержка). Эпик 12 (zero-click) и 15 (бот-участник конференций) — не MVP. Отложенное: ADR-019 (SMTP от общего адреса) и нарезка диаризации для длинных записей в пайплайне.
