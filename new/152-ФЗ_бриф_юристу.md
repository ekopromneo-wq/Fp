# 152-ФЗ — бриф для юриста (Gate §6.1 / §6.2)

Единственный внешний блокер публичного запуска Stenogram. Техническая
инфраструктура согласий и удаления **построена и работает** — нужны юридические
решения и утверждённые формулировки. Ниже: что уже есть в системе, и точные
вопросы, каждый с привязкой к месту в коде/конфиге, куда пойдёт ответ.

Не юридический документ — структура вопросов и точки применения ответов.

## Что за данные

Персональный сервис: пользователь записывает/загружает аудио встреч →
автоматическая транскрипция с разделением по голосам (диаризация) → протокол и
задачи → отправка в Email / Telegram / Битрикс24. Обрабатываются: аудио встреч,
тексты стенограмм, имена/контакты участников, голоса (для диаризации).

## Что технически уже готово (менять не нужно, только наполнить смыслом)

- **Аудит согласий** — таблица `consent_events`: неизменяемый лог grant/decline/
  revoke со снимком точного текста, версией, ip/ua. Доказывает ровно то, что
  видел пользователь.
- **Версионирование текстов** — `backend/src/consentTexts.js`,
  `CONSENT_TEXT_VERSION = '2026-07-17.draft1'`. Тексты сейчас — **заготовки**.
- **Отдельные согласия** (4 типа): `recording`, `transcription`, `ai_analysis`,
  `cross_border` + предупреждение перед КАЖДОЙ записью (снимок в
  `participant_notice`).
- **Удаление** — каскадное (проверено), аудио из хранилища удаляется; **бэкапы**
  БД/MinIO с ротацией `RETENTION_DAYS = 14` (`deploy/backup.sh`).
- **Доступ поддержки** — только по явному согласию пользователя.

## Вопросы к юристу

| # | Вопрос | Куда идёт ответ |
|---|--------|-----------------|
| 1 | **Оператор ПДн** — кто оператор (юрлицо/ИП), реквизиты, политика обработки ПДн (ссылка/документ). | Тексты согласий + политика в приложении. |
| 2 | **Тексты 4 согласий** — утвердить формулировки для `recording` / `transcription` / `ai_analysis` / `cross_border` + текст предупреждения перед записью. | `consentTexts.js`, затем **поднять `CONSENT_TEXT_VERSION`** (старые снимки в аудите сохраняются). |
| 3 | **Трансграничная передача** — процедура и правомерность. Фактический стек обработки (см. ниже) включает провайдеров в США. Нужно ли отдельное согласие/уведомление, на каких основаниях. | Текст `cross_border` + возможная смена набора провайдеров. |
| 4 | **Перечень AI-провайдеров и стран** — подтвердить/ограничить список (см. ниже), указать в согласии. | `cross_border` текст; при ограничении — конфиг моделей/провайдеров. |
| 5 | **Срок хранения и удаления из бэкапов** — за какой срок ПДн должны исчезать из резервных копий. | `RETENTION_DAYS` в `deploy/backup.sh` (сейчас 14 дней). |
| 6 | **Уведомление участников** — достаточно ли согласия владельца встречи + предупреждения, или нужно уведомлять/получать согласие самих участников, и как это доказывать. | Логика `ConsentDialog` + `participant_notice`; возможен новый поток. |
| 7 | **Голосовая биометрия** — является ли диаризация по голосу (кластеризация голосов; Speech2Text отдаёт голосовые векторы — сейчас не используются) обработкой биометрических ПДн, требуется ли отдельное согласие. | Возможно новый тип согласия + текст. |
| 8 | **Хардгейт обработки** — блокировать ли обработку записи без согласий (сейчас сознательно НЕ блокируется, чтобы не ломать существующие записи). | Гейт в конвейере обработки (worker) — если потребуется. |

## Фактический стек обработки (для вопросов 3–4)

| Провайдер | Что делает | Юрисдикция |
|-----------|-----------|------------|
| OpenRouter → OpenAI Whisper | распознавание речи (ASR) | США |
| OpenRouter → Google Gemini | диаризация по голосу | США |
| OpenRouter → Anthropic Claude | имена спикеров, протокол, задачи | США |
| Shopot (`api.shopot.ai`) | облачная диаризация | РФ |
| Speech2Text (`speech2text.ru`) | распознавание + бот-участник | РФ |

Пользователь выбирает метод диаризации в настройках; ASR/протокол/задачи по
умолчанию идут через OpenRouter (США). Полностью РФ-контур возможен (Shopot/
Speech2Text), но имена спикеров/протокол/задачи сейчас завязаны на Claude (США) —
это тоже вопрос к согласованию, если нужен only-РФ режим.

## После ответов — точки изменений в коде

1. `consentTexts.js` — вписать утверждённые тексты + **поднять `CONSENT_TEXT_VERSION`**.
2. `deploy/backup.sh` — выставить `RETENTION_DAYS` по сроку из п. 5.
3. При необходимости: новый тип согласия (голосовая биометрия, п. 7), хардгейт
   обработки (п. 8), only-РФ конфиг провайдеров (п. 4).

После этого §6.1 и §6.2 Gate закрываются.
