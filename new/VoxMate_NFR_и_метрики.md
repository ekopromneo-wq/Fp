### Смысл «результата через одну минуту»
Вы говорили, что через минуту после встречи пользователь получает «протокол и список задач». На практике за минуту реалистично гарантировать только: **запись сохранена, не потеряется, обработка началась, виден статус**. Готовый протокол за минуту — не SLA (зависит от длины и ASR).
⚠️ **Требует решения:** согласны переформулировать «минутную» цель как «сохранено + обработка началась», а готовый протокол оставить целью без жёсткого срока?

## 4. Качество транскрипции (ASR)

Вы задали «не менее 90% текста». Предлагаю разбить по типам аудио, потому что 90% на шумной записи недостижимо, а на чистой — заниженно:

| Тип аудио | Цель распознавания | Статус |
|-----------|-------------------:|--------|
| Чистая речь | ≥ 90% | ✅ ваша цифра |
| Переговорная, 2–6 спикеров | ≥ 85% | ⚠️ предложение |
| Шумная запись | ≥ 75% | ⚠️ предложение |
| Потерянные фрагменты | < 2% длительности | ⚠️ предложение |

Критичны: **имена, числа, даты, сроки, отрицания** (от них зависит корректность задач).
⚠️ **Требует решения:** принимаем разбивку по типам или держим единый порог 90%?

## 5. Качество извлечения задач

Ваш принцип: «хуже создать ложную задачу» → приоритет **precision**.

| Метрика | Цель (предложение) | Статус |
|---------|-------------------:|--------|
| Precision | ≥ 0,90 | ✅ утверждено (ADR-031) |
| Recall | ≥ 0,85 | ✅ утверждено |
| F1 | ≥ 0,87 | ✅ производная |
| Исполнитель при явном имени | ≥ 0,90 | цель, проверяется замером |
| Срок при явной дате | ≥ 0,95 | цель, проверяется замером |
| Относительная дата корректна | ≥ 0,90 | цель, проверяется замером |
| Задачи без ручной правки описания | ≥ 70% | цель, проверяется замером |
| Ложные задачи среди подтверждённых | < 5% | цель, проверяется замером |

Целевые метрики **утверждены** (precision ≥ 0,90 / recall ≥ 0,85, приоритет precision — «хуже создать ложную задачу»). Это цель, которая **подтверждается замером на реальном датасете (§8)**: до первого замера цифры не гарантируются, а служат критерием приёмки.

## 6. Неполные задачи — правило реализации

Вы разрешили отправлять задачи с «?» в незаполненных полях. Это создаёт риск, поэтому нужно чёткое правило:

1. В VoxMate неполную задачу **можно** подтвердить и отправить.
2. Перед отправкой — **предупреждение**.
3. В Email и Telegram «?» передаётся как **«Не определено»**.
4. В Битрикс24: если API требует `RESPONSIBLE_ID`, пользователь обязан выбрать ответственного, **иначе задача в Б24 не создаётся** (в Email/Telegram при этом уйти может).
5. Отслеживаем отдельной метрикой `incomplete_task_sent_rate`.

**Аналитические события:** `incomplete_task_confirmed`, `incomplete_task_sent_email`, `incomplete_task_sent_telegram`, `incomplete_task_blocked_b24`.

## 7. Качество протокола

Оценка 1–5 по критериям: соответствие темам, полнота решений, полнота задач, отсутствие вымышленных фактов, полезность без переработки.

Цели (⚠️ предложения):
- средняя оценка ≥ 4,0;
- ≥ 70% протоколов требуют только косметических правок;
- опасные вымышленные утверждения < 1%;
- AI-рекомендации визуально отделены от содержания встречи.

## 8. Эталонный датасет для приёмки AI

✅ Утверждено (ADR-032):
- **50 реальных встреч**, не менее **40 часов** суммарно;
- минимум **500 размеченных задач**;
- покрытие всех системных шаблонов протокола;
- **2 разметчика + арбитр** для спорных случаев.

**Повторная приёмка обязательна** при смене: ASR-модели, LLM, промптов, логики чанкинга/map-reduce, языка, шаблона или логики разрешения сроков.

## 9. Продуктовые метрики

- Activation rate (первый протокол получен + задачи открыты).
- Time to processing started / Time to first protocol.
- Second meeting rate (14 дней).
- Protocol acceptance rate / Task confirmation rate.
- Incomplete task sent rate / No-edit task rate.
- Delivery success rate по каждому каналу (Email/Telegram/Б24).
- Offline recovery rate.
- Zero-click rate.
- Retention D14.

**Аналитика не содержит** аудио, текста встреч, имён, контактов ✅ (152-ФЗ).

## 10. Безопасность и приватность

- TLS 1.3; шифрование at rest (AES-256) ✅ техспека.
- Signed URL с коротким TTL для аудио; токены — только в secret storage.
- Server-side ACL; audit log всех действий (вы подтвердили журнал действий).
- Поддержка не видит содержимое без согласия пользователя ✅.
- Rate limit; idempotency keys для воркеров.
- AI-оригинал стенограммы — **immutable** (сохраняется отдельно) ✅.
- Удаление проекта не удаляет встречи — они переходят в «Без проекта» ✅ (ADR-030).
- Корзина — 7 дней ✅.
- Голосовые шаблоны (биометрия) не хранятся в MVP ✅.

**Удаление проекта** (ADR-030): встречи не теряются — переходят в «Без проекта». Сам проект удаляется, история встреч сохраняется.
